Create the directory `.claude/commands/team-health/` if it does not exist. Then create `setup.md` with the following exact front matter and prompt body.

**Front matter** (use the exact verified keys from research):

```yaml
---
description: "First-time setup for team-health skill. Collects team roster, 1:1 cadence, and detects available MCP sources. Run once per manager before using any other team-health command."
argument-hint: ""
allowed-tools: Read, Write, Bash(git status), Bash(cat .gitignore), Bash(echo)
disable-model-invocation: true
---
```

**Prompt body** - write the full body as executable Claude instructions, not prose about instructions:

The body must cover these steps in order:

**Step 0 - Already set up check:**
Read `.team-health/config.json`. If it exists and `setup_complete` is true, output: "Team Health is already configured. Your team has [N] members. Sources detected: [list]. Run `/team-health:setup --reset` to reconfigure." Then stop.

If the user passed `--reset` as an argument (check `$ARGUMENTS`), proceed to step 1 even if already configured.

**Step 1 - MCP availability probe:**
Attempt to detect each MCP source by trying a minimal tool call for each. Because MCP tool names vary by implementation, instruct Claude to:

For GitHub: Try calling any tool whose name contains "github" or starts with "mcp_github". If any such tool is available and responds (even with an error that is not "tool not found"), set github = true. Otherwise set github = false.

For Jira: Same pattern - look for any tool whose name contains "jira". Set jira accordingly.

For Slack: Same pattern - look for any tool whose name contains "slack". Set slack accordingly.

For Calendar: Same pattern - look for any tool whose name contains "gcal", "google_calendar", or "calendar". Set calendar accordingly.

After probing all four, present a capabilities summary before asking any questions:
```
MCP Sources Detected:
- GitHub: [available / NOT AVAILABLE - PR/commit signals will be absent]
- Jira: [available / NOT AVAILABLE - ticket velocity signals will be absent]
- Slack: [available / NOT AVAILABLE - participation metadata signals will be absent]
- Calendar: [available / NOT AVAILABLE - meeting load signals will be absent]
```

**Step 2 - Collect manager info:**
Ask (one message, all at once - do not ask one field at a time):
- Your name
- Your GitHub username
- Your timezone (e.g. America/New_York) - optional, defaults to UTC
- Your GitHub organization or username for the team repos

**Step 3 - Collect team roster:**
Ask the manager to list their direct reports. For each person, collect:
- Full name
- Role
- GitHub username (optional)
- Default 1:1 cadence in weeks (default: 1)

Derive slug from name: lowercase, spaces to hyphens, remove special chars (e.g. "Alice Chen" → "alice-chen").

Confirm the roster before writing: "I'll set up team-health for [N] direct reports: [list names]. Is this correct?"

**Step 4 - Gitignore check:**
Before writing any files, read `.gitignore` using the Read tool.
- If `.team-health/` is already present, note it and continue.
- If not present (or file doesn't exist), use Bash to append `.team-health/` to `.gitignore`: `echo ".team-health/" >> .gitignore`
- Tell the user: ".team-health/ added to .gitignore to keep people data out of version control."

**Step 5 - Write config.json:**
Write `.team-health/config.json` with the exact schema shape from the canonical schema (schema_version "1"). Fields:
- schema_version: "1"
- setup_complete: true
- created: today's date (ISO format YYYY-MM-DD)
- last_updated: today's date
- manager: { name, github_username, timezone }
- team: array of team members with slugs derived in Step 3
- sources: the detected booleans from Step 1
- baseline_window_weeks: 8
- github_org: from Step 2
- sprint_cadence_weeks: 2
- last_skip_level: null

Optional fields that were not provided (jira_user_id, slack_user_id, start_date) should be set to null.

**Step 6 - Confirm and summarize:**
After writing config.json, output a confirmation:
```
Team Health setup complete.

Team roster saved: [N] people
MCP sources: GitHub [yes/no], Jira [yes/no], Slack [yes/no], Calendar [yes/no]

Next steps:
- Run /team-health:log <name> to start keeping notes about your team
- Run /team-health:pulse to run your first team health scan (requires GitHub MCP)
- Run /team-health:prep <name> before your next 1:1

Config saved to .team-health/config.json
.team-health/ is gitignored - your people data stays local.
```
