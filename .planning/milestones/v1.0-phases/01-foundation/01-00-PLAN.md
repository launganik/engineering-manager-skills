### Scenario 1 - MCP detection: no MCPs configured (SETUP-01)
**Setup:** Ensure no GitHub, Jira, Slack, or Calendar MCP servers are configured in Claude Code.
**Action:** Run `/team-health:setup` and complete the setup flow.

### Scenario 2 - MCP detection: GitHub MCP configured (SETUP-01)
**Setup:** Configure GitHub MCP in Claude Code (`claude mcp add github ...`). No other MCPs.
**Action:** Run `/team-health:setup` (or `--reset` if already run).
**Expected:** sources.github = true; remaining three = false.
**Pass criteria:** `cat .team-health/config.json` shows `"github": true`

### Scenario 3 - Setup writes correct config.json (SETUP-02)
**Setup:** Fresh project, no `.team-health/` directory.
**Action:** Run `/team-health:setup`. When prompted: add 2 team members (name, role, cadence).
**Expected:** `.team-health/config.json` exists with schema_version "1", setup_complete true, team array with 2 entries each having slug, role, and 1on1_cadence_weeks fields.
**Pass criteria:** JSON is valid; required fields present per state-schemas.json

### Scenario 4 - Setup gate routes to setup (SETUP-03)
**Setup:** Delete or rename `.team-health/config.json` so it does not exist.
**Action:** Run `/team-health:prep alice` (or any team-health command other than setup).
**Expected:** Command does NOT attempt to generate prep content. Instead it outputs: "Team Health is not set up yet. Run /team-health:setup to configure your team roster and MCP sources."
**Pass criteria:** No prep content generated; setup message displayed; command stops.

### Scenario 5 - All state files have schema_version (SETUP-04)
**Setup:** Complete setup (Scenario 3). Run `/team-health:pulse` once to generate baselines.json. Run `/team-health:log alice` to generate people/alice-<slug>.json.
**Action:** Inspect all three state files.
**Expected:** Each file has a top-level `schema_version` field set to "1".
**Pass criteria:** `cat .team-health/config.json | grep schema_version` returns `"schema_version": "1"` (same for baselines.json and people/<slug>.json)

### Scenario 6 - .team-health/ is gitignored (SETUP-05)
**Setup:** Fresh git-initialized project. Install skill files. Run setup.
**Action:** Run `git status` in the project directory.
**Expected:** `.team-health/` does not appear as an untracked directory. `.gitignore` contains `.team-health/`.
**Pass criteria:** `cat .gitignore | grep .team-health` returns `.team-health/`; `git status` does not list `.team-health/`

**Action:** Read `.claude/team-health/SIGNALS.md`.
**Expected:** File exists. Contains sections for: GitHub signals (PR frequency, review participation, commit days), Jira signals (tickets closed, blockers), Slack signals (channel participation, response latency), Calendar signals (meeting load, 1:1 adherence). Each signal includes: how it is computed, what data source it uses, and threshold guidance.
**Pass criteria:** Human reviewer confirms all signal categories are present and computation methodology is described.

### Scenario 8 - BASELINES.md exists and is substantive (REF-02)
**Setup:** Skill files installed.
**Action:** Read `.claude/team-health/BASELINES.md`.
**Expected:** File exists. Describes rolling 8-week window computation, mean and stddev calculation steps (inline arithmetic instructions for Claude), and the two-sigma flag threshold. Includes a worked example.
**Pass criteria:** Human reviewer confirms a Claude instance could follow the instructions to compute a baseline without additional context.

### Scenario 9 - PRIVACY.md exists and is substantive (REF-03)
**Setup:** Skill files installed.
**Action:** Read `.claude/team-health/PRIVACY.md`.
**Expected:** File exists. Contains explicit rules for: using behavioral observation language (not diagnostic), excluding DM content, not using team-relative scoring, not using psychological labels. Includes approved phrasing examples and prohibited phrasing examples.
**Pass criteria:** Human reviewer confirms all four privacy categories from REQUIREMENTS.md are covered with concrete examples.

### Scenario 10 - SKILL.md installation guide is accurate (REF-04)
**Setup:** Read top-level SKILL.md.
**Action:** Follow the installation steps exactly in a fresh terminal session.
**Expected:** After following the guide, `/team-health:setup` is available in Claude Code and the first-run flow can be completed.
